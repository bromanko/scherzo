# Clean up subprocesses on timeout

When a command timeout occurs in process.gleam:223-233, the spawned subprocess may continue running, causing resource leaks. Implement proper cleanup using OS-level signals (SIGTERM/SIGKILL) via Erlang's port mechanism to terminate the subprocess when a timeout is reached.
