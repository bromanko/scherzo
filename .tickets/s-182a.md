# Add filtering flags to tasks command

Add command-line flags to filter tasks output:

Flags:
- `--all` - Show all tasks including completed (overrides default)
- `--pending` - Show only pending/ready tasks
- `--in-progress` - Show only in-progress tasks
- `--status=<status>` - Filter by specific status
- `--tag=<tag>` - Filter by tag (e.g., `--tag=phase-5`)

Implementation:
- Update CLI argument parsing in `src/scherzo.gleam` for tasks command
- Pass filter options to `get_tasks()` function
- Add `TasksFilter` type to represent filter criteria in `commands.gleam`
- Apply filters before grouping/display
- Update `get_tasks()` signature to accept optional `TasksFilter`

REPL support:
- Update `tasks_command()` in `commands.gleam` to parse args (currently ignored)
- Support same filters in REPL: `tasks --pending`, `tasks --all`, etc.
- Parse args list into `TasksFilter` struct

Default behavior: Show pending + in-progress + failed (actionable tasks).
