# scherzo console CLI command

Implement 'scherzo console' CLI command as the primary entry point for the tmux UI.

Behavior:
- Check if tmux session 'scherzo' already exists
- If exists: attach to it (like 'scherzo attach')
- If not: create session via runner.start_with_session() and run REPL

This unifies session creation and attachment into a single user-friendly command. The existing 'repl' command remains for standalone/testing use (no tmux session management).

Implementation:
- Add console_command() function in src/scherzo.gleam
- Register at path ['console'] in glint router
- Reuse tmux.session_exists(), tmux.attach_session()
- Call runner.start_with_session() for new sessions

See PLAN.md Phase 4, item 37.
