# Parse parent field from ticket frontmatter

Add support for parsing the `parent` field from ticket frontmatter to enable hierarchical task views.

Current state:
- `ParsedTicket` type in `parser.gleam` lacks `parent` field
- `Task` type in `task.gleam` lacks `parent` field
- Frontmatter parser ignores `parent:` lines

Implementation:
1. Add `parent: Option(String)` to `ParsedTicket` type in `src/scherzo/task/sources/parser.gleam`
2. Add `parent` case to `parse_frontmatter()` function
3. Add `parent: Option(Id)` to `Task` type in `src/scherzo/core/task.gleam`
4. Update `ticket_to_task()` in `ticket.gleam` to map the parent field

This is a prerequisite for the tree view feature (s-4f6f).
